New Scheduler - OSU

Suppose you are asked to redesign the OSU course search page for graduate students who need to decide which courses to take in the next term. You've done interviews and learned that they want to see how their weekly schedule would look like while searching for courses.

From the previous scheduler from OSU, make changes in the design using design guidelines to make it more friendly

Low-Fidelity
What to improve?

Search for classes

Hover to see in calendar

Click to add to calendar

See conflicts and remove classes

High-Fidelity

Consistency applied by filtering elements are aligned and have the same color and format.

Grouped classes based on department and give feedback by adding their full name.

Deleting "X" from the paper prototype when a class is selected. The color code is intuitive enough.

Visibility of system status by saying how to interact and select classes.

Recover and prevention when adding more classes.

Recognition by keeping the elements in the same screen and adding arrows for flow.
